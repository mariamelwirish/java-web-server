This landing page is served directly by our custom Java HTTP server. Below are three test cases that demonstrate how our server handles different types of GET requests.

Test Files

Click to download test1.txt

We intentionally left test1.txt without a MIME mapping. The server falls back to application/octet-stream, causing the browser to download it instead of displaying it. This demonstrates the default behavior.

Click to open test2.html

test2.html loads normally because the server recognizes .html files and serves them using the text/html MIME type.

Click to request a missing file

This file does not exist. The server correctly returns a 404 Not Found response and displays our custom error page.
